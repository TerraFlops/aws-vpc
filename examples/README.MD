# Example VPC with Multiple Subnets

Refer to `main.tf` file in this folder for an example using this module to create the following
resources:

* VPC 10.0.0.0/18 - Example (Example Terraflops VPC)
    * DNS Support Enabled
    * DNS Hostnames Enabled
    * IPv6 Disabled
    * Internet Gateway
    * Security Groups
        * Application
            * Allow all inbound traffic from Application Subnet 2A
            * Allow all inbound traffic from Application Subnet 2B
            * Allow all outbound traffic to the VPC
            * Allow inbound MySQL traffic on TCP port 3306 from the "Database" security group
        * Database
            * Allow inbound MySQL traffic on TCP port 3306 from Application Subnet 2A
            * Allow inbound MySQL traffic on TCP port 3306 from Application Subnet 2B
            * Allow all outbound traffic 
        * NatInstance
            * Rules automatically generated by module
    * Availability Zone 2A
        * Public Subnet 2A (10.0.10.0/24)
            * EC2 NAT Instance + EIP
            * Default Route -> Internet Gateway
        * Application Subnet 2A (10.0.20.0/24)
            * Default Route -> EC2 Nat Instance
        * Data Subnet 2A (10.0.30.0/24)
            * Default Route -> EC2 Nat Instance
    * Availability Zone 2B 
        * Public Subnet 2B (10.0.11.0/24)
            * EC2 NAT Instance + EIP
            * Default Route -> Internet Gateway
        * Application Subnet 2B (10.0.21.0/24)
            * Default Route -> EC2 Nat Instance
        * Data Subnet 2B (10.0.31.0/24)
            * Default Route -> EC2 Nat Instance

### Disclaimer

Note: This is by no means an example of best practices- and should be treated purely as an example
of using this Terraform module.
